# fluter_lubs
Приложение для телефона "Трекер привычек и идей" 

🚀 Концепция приложения «Трекер привычек и идей»

Вместо списка повседневных задач, приложение будет помогать пользователю формировать полезные привычки и записывать вдохновляющие идеи. Сохранив лучшие архитектурные решения из исходного проекта, мы наполним его новым смыслом и взаимодействием.

🧱 Сравнение функциональности и структуры
Компонент	Исходное приложение (To-Do List)	Новое приложение (Habit & Idea Tracker)
Основная сущность	Задача (Task) с признаком выполнения.	Привычка (Habit) для регулярного повторения и Идея (Idea) для быстрых заметок.
Главный экран	Список задач + блок погоды.	Лента вдохновения: микс из целей на день (привычки) и последних идей.
Второй экран	"О приложении" (статичная информация).	"Конструктор привычек": экран для создания и настройки новых привычек (выбор цвета, иконки, периодичности).
Добавление записи	Единое поле ввода для задачи.	Две кнопки действий: «+ Привычка» (ведет в Конструктор) и «+ Идея» (вызывает диалог для быстрого ввода текста идеи).
Визуальное отличие	Чекбокс для выполнения.	Привычки отображаются с прогресс-баром (например, сколько дней подряд выполнена), Идеи — со стикером 💡 и датой.
Локальное хранение	Список задач.	Список привычек, список идей, настройки привычек.

🎯 Сценарии использования в новом приложении

1.Сценарий: Добавление новой привычки
Пользователь нажимает «+ Привычка» → попадает в «Конструктор привычек» → вводит название ("Пить воду"), выбирает иконку (💧), цвет (синий) и периодичность ("Ежедневно") → нажимает "Сохранить". Новая привычка появляется на главном экране.

2.Сценарий: Отметка выполнения привычки
На главном экране пользователь видит привычку "Пить воду" с прогресс-баром. Он нажимает на круглую кнопку "Выполнить" рядом с ней. Прогресс-бар заполняется на одну единицу, а система поощряет пользователя всплывающим уведомлением ("Отлично! День 1 из 7").

3.Сценарий: Быстрая запись идеи
Пользователя внезапно посещает мысль. Он нажимает кнопку «+ Идея» → появляется диалоговое окно → вводит текст ("Купить книгу по Flutter") → нажимает "Сохранить". Идея сразу появляется в ленте как карточка с датой и стикером 💡.

4.Сценарий: Просмотр статистики (вместо погоды)
На главном экране, там где в исходном приложении была погода, теперь отображается небольшой блок "Прогресс дня": "Сегодня выполнено: 3 из 5 привычек" и круговая диаграмма. Это мотивирует пользователя, заменяя внешнюю информацию (погода) на внутреннюю (прогресс).

📝 Технические заметки для реализации

Навигация: Можно оставить BottomNavigationBar, но изменить пункты на: «Лента» и «Конструктор привычек».

Хранение данных: Так же использовать shared_preferences или sqflite для хранения двух типов объектов (Habit и Idea), а также записей о выполнении привычек для прогресс-бара.

Состояние: Управление состоянием (например, с помощью Provider) усложнится, так как потребуется обновлять прогресс привычек и фильтровать идеи.
